# 第 01 章：依赖与版本基线

## 本章目标

先把“能不能装、能不能跑、后面会不会版本打架”这件事解决掉。

这一章结束后，你应该得到三件东西：

1. `pyproject.toml` 里已经准备好 LangChain / LangGraph 依赖
2. 配置层能表达新的模型初始化方式
3. 你知道后面哪些旧抽象会被逐步退休

## 本章要改哪些文件

- `pyproject.toml`
- `ipilot/config/schema.py`
- `ipilot/runtime.py`

这一章不要改 `ipilot/agent/loop.py`。主循环放到第 5 章再动。

## 1. 先加依赖

建议按下面的固定版本写，不要追最新。

```toml
[project]
dependencies = [
    "croniter>=6.2.2",
    "fastapi>=0.135.3",
    "httpx>=0.28.1",
    "openai>=2.30.0",
    "pydantic>=2.12.5",
    "pytest>=9.0.2",
    "pytest-asyncio>=0.26.0",
    "typer>=0.12.0",
    "uvicorn>=0.43.0",
    "websocket>=0.2.1",
    "websockets>=15.0.1",
    "langchain==1.2.15",
    "langgraph==1.1.6",
    "langchain-openai==1.1.8",
    "langgraph-checkpoint==4.0.1",
    "langgraph-checkpoint-sqlite==3.0.3",
]
```

为什么先不删 `openai`：

- 当前仓库已经在用
- 迁移过程中你可能还会暂时保留 `OpenAICompatibleProvider`
- 等第 5 章完成以后，再决定是否移除直接依赖

## 2. 配置结构先扩一层

当前 `ipilot/config/schema.py` 里的默认 agent 配置只知道：

- `workspace`
- `model`
- `provider`

迁到 LangChain 后，模型初始化通常还会依赖这些信息：

- provider 名称
- 模型名
- base URL
- API key
- 温度等模型参数

建议你把 schema 先改成下面这个方向：

```python
from pathlib import Path
from pydantic import BaseModel, Field


class AgentDefaults(BaseModel):
    workspace: str = "~/.ipilot/workspace"
    provider: str = "stepfun"
    model: str = "step-3.5-flash"
    temperature: float = 0.0


class ProviderConfig(BaseModel):
    api_key: str = ""
    api_base: str | None = None
```

这一步看起来变化不大，但后面很重要：

- 第 2 章会从这里读取 provider/model/api_base
- 第 5 章会把这些信息写进 `AgentRuntimeContext`

## 3. 在 `runtime.py` 里先建过渡层

当前 `ipilot/runtime.py` 还是这样：

- 读取配置
- 构造 `OpenAICompatibleProvider`
- 构造 `SessionManager`
- 构造 `ContextBuilder`
- 构造 `AgentLoop`

这条链会被分阶段拆掉。现在先不要一下删完，而是先让 `runtime.py` 多出几个新入口函数。

建议先加出这样的骨架：

```python
from ipilot.config.schema import Config


def build_chat_model(config: Config):
    raise NotImplementedError


def build_langchain_tools(config: Config):
    raise NotImplementedError


def build_agent_runtime_context(config: Config, *, session_key: str, channel: str | None, chat_id: str | None):
    raise NotImplementedError
```

这个阶段的目标不是“做完”，而是先把未来的装配点固定下来。

## 4. 这一章要明确退休哪些旧抽象

后面迁移时，这几个抽象会逐步退场：

- `LLMProvider`
- `OpenAICompatibleProvider`
- `ToolRegistry`
- `SessionManager`
- `ContextBuilder`

注意这里说的是“从主路径退场”，不是“立刻删除文件”。

## 5. 为什么这一章不能省

你现在这套仓库已经有：

- CLI
- SDK facade
- HTTP API
- heartbeat
- Twitch channel

如果不先把依赖、配置和 runtime 装配点固定住，后面你会同时改：

- 模型
- 工具
- 会话
- 图状态
- 入口层

那不是迁移，是自找冲突。

## 6. 本章常见坑

### 坑 1：直接删掉旧 provider

不要。第 2 章以前，它还是你的保底实现。

### 坑 2：依赖先写成宽松范围

不要。教程追求“跟着敲就能复现”，不是“以后大概能跑”。

### 坑 3：把 `session_key` 继续当普通字符串传来传去

后面它会成为 LangGraph 的 `thread_id`。从现在开始就要用这个视角看它。

## 7. 本章验证

完成后，至少跑下面几步：

```bash
uv sync
uv run python -c "import langchain, langgraph; print(langchain.__version__)"
uv run pytest -q tests/test_cli_commands.py tests/test_api_server.py
```

验收标准：

- `uv sync` 成功
- 能导入 `langchain`、`langgraph`
- 现有 CLI / API 基础测试没有因为依赖或配置层修改而先炸掉

如果这里还没过，不要进入第 2 章。
